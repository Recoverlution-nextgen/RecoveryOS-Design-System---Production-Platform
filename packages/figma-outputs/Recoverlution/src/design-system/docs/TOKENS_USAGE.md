Using generated tokens

Files generated by `npm run build-tokens` (Style Dictionary):
- `dist/tokens/css/variables.css` — CSS variables you can import globally.
- `dist/tokens/js/tokens.js` — CommonJS token module for runtime usage.
- `dist/tokens/json/tokens.json` — flat JSON for platform pipelines.
- `dist/tokens/android/colors.xml` — Android resource file.
- `dist/tokens/ios/Colors.swift` — simple Swift UIColor extensions for iOS handoff.

How to include in Storybook / app
- Import the generated CSS variables early in your app or Storybook preview: `import "../dist/tokens/css/variables.css";`.
- Use CSS variables in components: `color: var(--color-text-primary)` with fallbacks if needed.
- For JS usage, require the JS module: `const tokens = require('../dist/tokens/js/tokens.js')`.

Regeneration
- Run `npm run build-tokens` whenever `src/tokens/tokens.json` changes. CI runs this on push/PR.

Notes
- The generated CSS uses flat variable names (e.g. `--color-brand-primary`). Prefer these over raw hex in components.
- For platform-specific builds, use the provided Android XML and `Colors.swift` as starting points; you may refine formats to match your native coding conventions.
